# ADMIN_PLAN.md — Admin Panel Architecture

## Overview

Admin panel berbasis SSR di `/admin/*` dengan autentikasi Google OAuth melalui **Better Auth**.
Hanya email yang terdaftar di whitelist (`src/lib/auth-whitelist.js`) yang boleh masuk.

---

## Stack & Compatibility

| Layer | Pilihan | Alasan |
|---|---|---|
| Auth library | `better-auth` | First-class Astro integration, libsql adapter bawaan |
| OAuth provider | Google | Better Auth `socialProviders.google` |
| Session store | Turso (libsql) | DB sudah ada; Better Auth auto-create auth tables |
| Framework | Astro 6.1.3 + Netlify SSR | Routes admin = `prerender: false` |
| Styling | Design tokens existing | Konsisten dengan porto |

---

## Arsitektur Alur Auth

```
Browser
  │
  ├─ GET /admin/*
  │     └─ src/middleware.ts
  │           ├─ session valid? → render page
  │           └─ no session   → redirect /admin/login
  │
  ├─ GET /admin/login
  │     └─ tombol "Login with Google"
  │           └─ POST → /api/auth/sign-in/social (Better Auth)
  │
  ├─ GET /api/auth/[...all]   ← Better Auth handler (catch-all)
  │     ├─ /api/auth/sign-in/social     (mulai OAuth flow)
  │     ├─ /api/auth/callback/google    (Netlify function)
  │     └─ /api/auth/sign-out           (hapus session)
  │
  └─ Google OAuth Redirect
        └─ kembali ke /api/auth/callback/google
              ├─ email ada di whitelist? → set session cookie → /admin
              └─ email TIDAK di whitelist → redirect /admin/login?error=unauthorized
```

## Database — Tabel Auth (auto-created Better Auth)

Better Auth dengan libsql adapter akan membuat tabel berikut secara otomatis via migration:

```sql
-- auto-generated by better-auth
user        (id, name, email, emailVerified, image, createdAt, updatedAt)
session     (id, userId, expiresAt, token, ipAddress, userAgent, createdAt, updatedAt)
account     (id, userId, accountId, providerId, accessToken, refreshToken, ...)
verification (id, identifier, value, expiresAt, createdAt, updatedAt)
```

Tidak mengganggu tabel existing (`projects`, `writing`, `publications`, `speaking`).

## Whitelist Email

File `src/lib/auth-whitelist.js` berisi array plain JS:

```js
export const ALLOWED_EMAILS = [
  "hasban@example.com",
  // tambah email lain di sini
];
```

Digunakan di dua tempat:
1. **`src/lib/auth.ts`** — Better Auth `hooks.after` pada `signIn` untuk memblokir email di luar whitelist
2. **`src/middleware.ts`** — double-check pada setiap request ke `/admin/*`

## Environment Variables Baru

Tambahkan ke `.env` dan Netlify dashboard:

```env
# Better Auth
BETTER_AUTH_SECRET=random-32-char-string   # openssl rand -base64 32
BETTER_AUTH_URL=https://yourdomain.netlify.app

# Google OAuth (dari Google Cloud Console)
GOOGLE_CLIENT_ID=xxxx.apps.googleusercontent.com
GOOGLE_CLIENT_SECRET=GOCSPX-xxxx
```

Google Cloud Console setup:
- Authorized redirect URI: `https://yourdomain.netlify.app/api/auth/callback/google`
- Untuk dev: `http://localhost:4321/api/auth/callback/google`

## Security Design

| Concern | Solusi |
|---|---|
| Unauthorized access | `src/middleware.ts` — redirect semua `/admin/*` tanpa session valid |
| Email bukan whitelist | Better Auth `hooks.after.signIn` + middleware double-check |
| Session hijacking | Better Auth cookie `httpOnly: true, secure: true, sameSite: 'lax'` |
| CSRF | Better Auth menggunakan signed tokens bawaan |
| Secret leakage | `BETTER_AUTH_SECRET` hanya via `import.meta.env` server-side |
| Dev vs prod | URL dikontrol via `BETTER_AUTH_URL` env var |

## Admin Panel UI Design

Menggunakan design tokens yang sudah ada (`bg-card`, `border-border`, `text-foreground`, dll).
Tidak tambah dependensi UI baru.

Setiap halaman tabel menampilkan:
- Header dengan nama tabel + tombol "+ Add New"
- Tabel responsive (scroll horizontal di mobile)
- Tombol Edit (→ `/admin/{table}/{id}`) dan Delete (form POST inline)
- Form add/edit menggunakan field input yang sama dengan `contact.astro`

Layout admin menggunakan `AdminLayout.astro` tersendiri (tidak mewarisi `Layout.astro` publik) dengan:
- Sidebar navigasi antar tabel
- Tombol "Sign out" di pojok kanan atas
- Tidak ada Navbar publik
